# xUnitのテストが壊れたときのSkillを考える

xUnitでテストが壊れたとき、どのように修正すべきか考える。新規でテストを書くときは特定の条件のときの振る舞いを書くだけでよい。しかし、テストがパスしなくなったらどのように修正すればよいのかはテストのほうに問題があるのかコードのほうに問題があるのかテストを書いたことがない人間には判断が難しいときもある。(もちろん上級者でも難しい場合もある)

## テストとコードのどちらを修正すればよいか

テストのArrange,Action,Assertに間違いがなければコードが間違っている。リファクタリングに対して異常検知をできるのがユニットテスト良いとこである。つまり普段我々が書いているテストは凡そがリファクタリング耐性を上げるためにテストを書いていると言える。
